# ADR-0023：使用单页响应式界面和两次短暂上传

- 状态：已接受
- 日期：2026-07-23

## 背景

上传后需要读取 Excel 中的默认日期，但用户可能在点击处理前停留较长时间。产品同时要求后端不长期保存未处理文件。

## 决策

- 登录后使用单页响应式工具界面。
- 页面依次提供上传、股票代码、日期、三个独立复选框、处理按钮及进度消息。
- 文件选择后第一次上传用于模板分析，完成后后端立即删除分析副本。
- 浏览器页面内存保留原始 `File` 对象；点击处理时第二次上传并执行完整校验和回填。
- 页面刷新或关闭后需要重新选择文件。
- 成功后自动下载；自动下载受阻时提供下载按钮，文件只留在当前页面内存。
- 桌面端使用居中单列卡片，移动端使用全宽单列布局。

## 影响

- 同一个文件在一次完整操作中会上传两次，但文件很小且使用频率低。
- 后端无需在用户编辑表单期间长期保存 Excel。
- 前端必须在处理开始前确认所选文件对象仍然可用。
