# ADR-0022：日期统一使用严格点分格式

- 状态：已接受
- 日期：2026-07-23

## 背景

不同桌面和移动浏览器对原生日期控件的显示格式不同，而模板标题要求稳定使用 `yyyy.MM.dd`。

## 决策

- 日期使用文本框和日历按钮组合控件。
- 页面只接受并显示严格的 `yyyy.MM.dd`，不自动猜测斜杠、短横线或其他格式。
- 日期必须真实有效，开始日期不晚于结束日期。
- 修改后的开始、结束日期必须分别严格晚于上一时间段对应日期。
- 错误时显示字段提示并禁用处理按钮。
- 日期按不含时区的纯日期处理。
- 写回标题固定为 `yyyy.MM.dd - yyyy.MM.dd`。

## 影响

- 前端需要自定义日期格式化及日历选择体验。
- 后端仍须独立执行同样校验，不能只信任前端。
