# ADR-0021：使用结构化错误代码且不生成部分文件

- 状态：已接受
- 日期：2026-07-23

## 背景

产品只有一个进度区域，需要让个人用户能够理解模板、输入、行情或输出阶段发生的问题，同时不能暴露服务器内部信息。

## 决策

- 错误按 `AUTH`、`FILE`、`TEMPLATE`、`INPUT`、`MARKET`、`OUTPUT`、`SYSTEM` 分类并分配稳定编号。
- 用户错误信息包含失败阶段、股票代码、日期、字段、原因及未生成文件的结果说明。
- 错误内容支持一键复制。
- 预检查错误和多股票无行情错误尽量汇总展示。
- 网络临时错误完成最多 3 次重试后再报告最终错误。
- 页面不显示服务器路径、程序堆栈和接口原始响应。
- 任意错误均不生成部分输出文件。
- 服务端日志只记录任务编号、错误代码、阶段和耗时，不记录 Excel 正文。

## 影响

- 后端错误必须映射到稳定的业务错误对象，不能直接返回异常文本。
- 前端进度条在失败阶段停止并变为错误状态。
- 测试用例需要覆盖每个错误类别及其用户文案。
