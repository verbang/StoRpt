# ADR-0004：按严格时间顺序识别最新时间段

- 状态：已接受
- 日期：2026-07-23

## 背景

工具只能操作最新时间段，因此必须在任何写入前无歧义地识别目标区块。业务约定各时间段的开始、结束日期互不相同，并且越靠下的区块日期越新。

## 决策

1. 扫描并解析工作表中的全部时间段标题，标题格式为 `yyyy.MM.dd - yyyy.MM.dd`。
2. 每个时间段必须满足开始日期不晚于结束日期。
3. 从上到下，相邻时间段的开始日期和结束日期都必须严格递增。
4. 开始日期或结束日期不得重复。
5. 通过校验后，最下方区块即为最新时间段，同时也是开始、结束日期最大的区块。
6. 任一条件不满足时，将文件判定为不兼容格式，报告对应错误并停止处理。
7. 不兼容文件不进入行情查询和回填阶段，不生成新文件。
8. 用户在界面修改最新时间段的日期后，处理前必须对修改后的完整时间序列重新执行相同校验。

## 影响

- 最新时间段的识别同时依赖日期语义和区块位置，不会在二者冲突时猜测目标。
- 错误消息需要指出问题区块、单元格及错误类型，便于用户修复模板。
- 上传解析与兼容性校验必须在所有文件写入之前完成。
