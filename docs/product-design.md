# A 股历史价格回填工具 - 产品设计文档

> 状态：已确认（MVP 基线）  
> 版本：1.0  
> 更新日期：2026-07-23

## 1. 产品目标

为桌面端和移动端用户提供一个跨平台工具：用户提交与 `platform.xlsx` 结构兼容的交易记录文件及股票查询条件，系统获取中国 A 股历史价格，将结果写入最新时间段，并生成一个新文件。

## 2. 已确认需求

### 2.1 运行环境

- 桌面端：Windows、macOS、Linux。
- 移动端：iOS、Android、HarmonyOS。

### 2.2 核心能力

- 获取中国 A 股具体个股的历史价格。
- 读取 Excel 文件。
- 将行情结果写入 Excel 文件。
- 保留用户上传的原文件，输出一个另存文件。
- 输出文件保持上传文件的原格式：`.xls` 输入对应 `.xls` 输出，`.xlsx` 输入对应 `.xlsx` 输出。

### 2.3 必要界面元素

| 元素 | 类型 | 初始规则 |
| --- | --- | --- |
| 上传文件 | 按钮 | 仅接受 `.xls`、`.xlsx` |
| 股票代码 | 文本输入 | 至少 1 个；业务层不设置固定数量上限 |
| 开始时间 | 日期输入 | 展示和写入格式为 `yyyy.MM.dd` |
| 结束时间 | 日期输入 | 展示和写入格式为 `yyyy.MM.dd` |
| 是否填入股票名称 | 独立复选框 | 默认未勾选；未勾选时保留 B 列原值 |
| 是否填入理想进价 | 独立复选框 | 默认未勾选；未勾选时保留 C 列原值 |
| 是否填入理想出价 | 独立复选框 | 默认未勾选；未勾选时保留 D 列原值 |
| 处理 | 按钮 | 校验通过后启动行情获取、回填、自检和另存 |
| 进度 | 进度条及消息区域 | 展示处理进度；失败时展示对应错误 |

### 2.4 已确认主流程

1. 用户上传 Excel 文件。
2. 系统读取文件中最新时间段，将其开始日期和结束日期回填至界面。
3. 用户可以修改开始日期和结束日期；修改值最终写回输出文件的最新时间段标题。
4. 用户输入一个或多个中国 A 股代码。
5. 用户分别决定是否填入股票名称、理想进价和理想出价。
6. 用户点击“处理”。
7. 系统把股票代码依次写入最新时间段的“股票代码”列。
8. 若选择填入股票名称，系统查询并写入“股票名称”列。
9. 若选择填入理想进价，系统查询每只股票在开始时间的开盘价并写入“理想进价”列。
10. 若选择填入理想出价，系统查询每只股票在结束时间的收盘价并写入“理想出价”列。
11. 系统自检本次应处理字段是否完整且正确。
12. 自检通过后，以处理当天日期作为基础文件名生成同格式文件，通过浏览器下载并报告完成；失败则不报告成功，并在消息区域展示错误。

### 2.5 使用范围与规模

- 仅供个人使用，不对外提供公共或商业服务。
- 每周通常处理 1 至 2 次。
- 每次通常处理 1 至 2 个 Excel 文件。
- 每个文件通常不超过 10 只股票。
- 优先采用零成本数据方案，可以接受免费接口无服务等级保证及偶发不可用。

## 3. 模板契约

当前模板分析详见 [template-analysis.md](template-analysis.md)。已经确认的结构如下：

- 样表当前只有 `Sheet1`，有效范围为 `A1:S36`；正式工具不依赖固定工作表名称。
- 样表中的每个时间段占 9 行：日期标题 1 行、表头 1 行、股票数据 6 行、间隔 1 行。
- 日期标题合并 `A:S`，格式示例为 `2026.01.26 - 2026.01.30`。
- 当前共有 4 个时间段，位于第 1、10、19、28 行。
- 当前最新时间段为 `2026.01.26 - 2026.01.30`，股票数据区为第 30 至 35 行。
- 本工具直接写入的列为：A 股票代码、B 股票名称、C 理想进价、D 理想出价。
- E 至 S 列包含依赖 A 至 D 及人工交易数据的公式和衍生结果，属于严格禁止写入的区域。
- 股票代码单元格采用文本格式，必须保留前导零。

### 3.1 数据写入安全边界

以下规则是产品底线，不得由实现方案改变：

- 工具只允许修改最新时间段对应区域的 A 至 D 列。
- 工具不得写入、清空、移动或重建 E 至 S 列中的任何单元格。
- 工具不得插入或删除整行，因为这会间接移动或修改 E 至 S 列。
- 工具不得修改其他历史时间段的 A 至 D 列。
- 股票数据区不绑定固定行号或 6 行容量。工具动态识别最新时间段的数据起点，并根据本次输入数量确定连续目标区域。A 列始终更新；B 至 D 列按各自复选框决定是否写入。
- 最新时间段必须是工作表最后一个业务区块。其表头以下的 A 至 D 全部保留给该时间段的动态股票数据，不允许混入备注、汇总、第二张表、公式或合并单元格。
- 自检不得以修复为名修改边界外数据，也不得清空未勾选字段。

## 4. 领域对象

| 对象 | 最终定义 | 关键约束 |
| --- | --- | --- |
| 用户任务 | 一次文件分析、一个或多个代码查询、最新时间段回填、自检和下载过程 | 单文件不超过 10 MB；同一账号一次一个任务；总时限 180 秒 |
| 源 Excel | 与模板结构兼容且不可被覆盖的 `.xls` 或 `.xlsx` 文件 | 采用严格兼容校验；不支持特殊 Excel 功能 |
| 目标工作表 | 工作簿中唯一一个可见且结构兼容、允许处理最新时间段的工作表 | 必须恰好识别一个；名称不限 |
| 股票标的 | 上交所、深交所或北交所当前上市的 A 股股票 | 包含 `ST`、`*ST`；不含退市股及其他证券 |
| 查询区间 | 用户选择的开始日期至结束日期 | 使用纯日期；指定日无行情时不顺延 |
| 行情结果 | 股票名称及指定日期对应的不复权开盘价、收盘价 | 首版来自 AKShare 封装的东方财富数据 |
| 最新时间段 | 全部时间段通过严格递增校验后，位于最下方且日期最大的区块 | 是唯一允许修改 A 至 D 的业务区块 |
| 输出 Excel | 保持源文件格式、回填最新时间段后由浏览器下载的新文件 | 不覆盖源文件；浏览器决定实际保存位置 |

## 5. 最终业务边界

### 5.1 MVP 纳入范围

- 单用户密码登录和 7 天安全会话。
- `.xls`、`.xlsx` 单文件上传、模板分析、处理和下载。
- 唯一目标工作表及严格递增时间段识别。
- 用户编辑最新时间段的开始、结束日期并写回标题。
- 一个或多个沪深京当前上市 A 股代码校验和 A 列完整替换。
- 股票名称、开始日不复权开盘价、结束日不复权收盘价三个字段的独立可选回填。
- 仅对最新时间段 A 至 D 执行单元格级白名单写入；E 至 S 及其他区域保持不动。
- 保持输入工作簿格式，执行保存后重开和结构化自检。
- 分阶段进度、结构化错误、网络错误最多重试 3 次和原子失败。
- 桌面及移动浏览器响应式 PWA、自动下载与受阻后的手动下载。
- 单镜像 Docker 部署、HTTPS、临时文件隔离与无业务留存。

### 5.2 MVP 不纳入范围

- 原生 Windows、macOS、Linux、iOS、Android 或 HarmonyOS 客户端。
- 多用户、注册、角色权限、任务队列、批量文件处理和文件历史。
- 通用 Excel 编辑、自动迁移不兼容模板或修改非最新时间段。
- 对 E 至 S 的值、公式、格式、公式缓存或收益结果进行写入、修复、扩展或业务校验。
- 退市股票及 B 股、基金、ETF、指数、债券、可转债等非目标证券。
- 指定日期无行情时自动顺延或回退到其他交易日。
- 复权行情、实时行情、分时行情及交易功能。
- 百度搜索、浏览器模拟抓取、多个行情源静默切换或付费行情接入。
- 密码、保护、宏、签名、外链、数据连接、透视表、图表、图片、形状或嵌入对象的保真处理。
- 保证文件保存到上传文件的原目录；网页只能发起浏览器下载。

## 6. 非功能约束

- 个人低频使用：每周约 1 至 2 次，每次约 1 至 2 个文件，通常不超过 10 只股票。
- 单文件最大 10 MB；同一账号同时只允许一个任务；单任务总超时 180 秒。
- 行情网络类错误首次失败后最多重试 3 次，即最多尝试 4 次；确定性错误不重试。
- 上传、行情、输出和认证响应不得被 PWA 或中间缓存保存。
- 正常任务结束后立即清除临时文件；异常任务临时文件最长保留 10 分钟。
- 日志不记录 Excel 内容、股票交易数据、密码、会话信息、服务器路径或完整文件名。
- 首版可接受免费行情源无 SLA，但任何数据源失败都必须显式失败，不得静默换源。

## 8. 讨论记录

### 第 1 轮：Excel 的角色

已确认：股票代码由页面输入且允许多个；Excel 是结构兼容的交易记录模板；本次结果写入最新时间段的 A 至 D 列相关字段。

### 第 2 轮：单个时间段的股票容量

已确认：每次至少输入 1 个股票代码，业务层不设置固定数量上限。工具只允许操作最新时间段的 A 至 D 列；E 至 S 列不得被工具修改，这是硬性安全边界。

### 第 3 轮：A 至 D 的替换语义

已修正：用户输入的代码列表是最新时间段 A 列的完整新集合。工具先清除最新时间段原有 A 列代码，再从动态识别出的首个数据行开始连续写入本次代码。B 至 D 不执行无条件清空，而是遵守各自复选框规则。数据行数由本次输入数量决定，不绑定第 36 行或任何固定结束行。

工具不分析、不提示新 A 至 D 数据与 E 至 S 原有数据之间的业务关联。E 至 S 仅作为受保护区域：不得修改，但不纳入本次行情回填正确性的业务判断。

### 第 4 轮：最新时间段识别

已确认：每个时间段的开始日期和结束日期均应唯一，并且区块越靠下，开始日期和结束日期都应越新。工具先解析并校验全部时间段，再将最下方且日期最大的区块识别为最新时间段。

若任一标题格式错误、开始日期晚于结束日期、日期重复、开始日期未严格递增、结束日期未严格递增，或者日期顺序与区块位置不一致，则文件属于不兼容格式。工具报告具体错误并立即停止，不执行回填，也不生成新文件。用户在界面修改最新时间段后，处理前必须再次执行相同校验。

### 第 5 轮：无行情日期处理

已确认采用严格日期口径。勾选“填入理想进价”时，必须取得每只股票在开始日期当天的开盘价；勾选“填入理想出价”时，必须取得每只股票在结束日期当天的收盘价。

周末、休市、停牌或其他原因导致任意一只股票在指定日期没有所需行情时，整个任务失败。工具不得改用之前或之后的交易日，不得生成部分成功文件。未勾选的价格字段不触发对应日期的行情可用性检查。

### 第 6 轮：价格复权口径

已确认采用不复权价格。理想进价使用股票在指定开始日期的实际日线开盘价，理想出价使用股票在指定结束日期的实际日线收盘价，不进行前复权或后复权调整。

### 第 7 轮：股票范围与代码输入

已确认：支持上交所、深交所和北交所当前上市的 A 股，包括 `ST`、`*ST` 股票；不支持已退市股票、B 股、基金、ETF、指数、债券、可转债及其他非 A 股证券。

用户输入六位纯数字代码，多个代码可用空格、换行、英文逗号、中文逗号或分号分隔。系统清理分隔符后保留输入顺序；重复代码、格式错误、不存在或不属于支持范围的代码均导致整个任务停止，不回填、不生成文件。证券类型和上市状态必须通过当前 A 股证券清单校验，不得只依赖代码前缀判断。

### 第 8 轮：可选字段控件与未勾选行为

已确认：“股票名称”“理想进价”“理想出价”是三个互相独立的复选框，不是互斥单选项。三个复选框默认均未勾选，允许任意组合。

- A 列股票代码始终按本次输入列表更新。
- 勾选股票名称时写入 B 列；未勾选时不写 B 列、不清空原值。
- 勾选理想进价时写入 C 列；未勾选时不写 C 列、不清空原值。
- 勾选理想出价时写入 D 列；未勾选时不写 D 列、不清空原值。
- 三项均未勾选时，仍校验股票代码，并只执行日期及 A 列代码的允许操作。

### 第 9 轮：未勾选字段的原值归属

已确认未勾选的 B、C、D 按单元格位置保留，不按股票代码匹配：

- A 列代码发生变化时，同一行未勾选的 B 至 D 保持原值。
- 本次代码数量少于原代码数量时，尾部旧 A 列代码被清除，但同一行未勾选的 B 至 D 仍保持原值。
- 工具不移动、复制、清空或重新匹配这些原值，也不提示其与当前 A 列代码的对应关系。

### 第 10 轮：动态数据区边界

已确认：最新时间段必须是工作表最后一个业务区块。工具将该时间段表头的下一行作为动态数据起点，结束行由本次股票代码数量决定，不使用固定行号或固定容量。

- 工具不插入或删除整行，只操作目标 A 至 D 单元格。
- 最新时间段表头以下的 A 至 D 只能包含该时间段股票数据。
- A 至 D 下方不得存在备注、汇总、第二张表或其他业务内容。
- 发现 A 至 D 下方存在公式、合并单元格或可识别的其他业务内容时，判定模板不兼容，停止处理且不生成文件。
- E 至 S 不参与动态边界判断，并始终保持受保护状态。

### 第 11 轮：输出文件格式

已确认输出文件保持上传文件的原格式：

- 上传 `.xls` 时输出 `.xls`。
- 上传 `.xlsx` 时输出 `.xlsx`。
- 不进行 `.xlsx` 与 `.xls` 之间的格式转换。
- 除本次明确授权的单元格操作外，不随意改变原有公式、样式、合并关系或其他工作簿结构。
- 文件格式自身的行数限制仍然有效，不视为产品设置的股票数量业务上限。

### 第 12 轮：跨平台交付与保存位置

已确认采用统一网页/PWA：

- Windows、macOS、Linux、iOS、Android 和 HarmonyOS 均通过浏览器使用。
- 处理完成后由浏览器下载输出文件。
- 工具不保证保存至上传文件同目录；保存位置由浏览器或操作系统决定。
- 输出文件以北京时间处理当天的 `yyyyMMdd` 作为基础名称，并保持输入扩展名。
- 不覆盖用户已有文件。

同名编号按当前浏览器保存的应用生成记录计算：当天首次使用基础名称，之后依次追加 `_01`、`_02`。更换浏览器、设备或清除浏览器数据后计数重新开始；若下载目录仍存在同名文件，浏览器可以再次重命名。工具本身不得覆盖已有文件。

### 第 13 轮：目标工作表识别

已确认通过结构识别目标工作表，不依赖 `Sheet1` 等固定名称：

- 扫描所有可见工作表。
- 候选表必须包含可解析的时间段标题、依次位于 A 至 D 的“股票代码、股票名称、理想进价、理想出价”表头，并通过时间段严格递增校验。
- 恰好找到一个候选表时，将其作为目标工作表。
- 没有候选表或存在两个及以上候选表时，报告对应不兼容错误并停止处理，不生成文件。
- 隐藏工作表不能作为目标工作表。
- 其他可见或隐藏工作表保持原样，不参与业务处理。

### 第 14 轮：Excel 功能兼容边界

已确认采用严格模板兼容模式。允许并要求保留普通单元格值、公式、字体、颜色、边框、填充、对齐、数字格式、合并单元格、行高、列宽、工作表名称及顺序、显示状态、冻结窗格和基础打印设置。

存在以下任一功能时，文件判定为暂不兼容，工具报告具体功能并停止处理，不生成输出文件：

- 文件加密或需要密码。
- 工作簿结构保护或目标工作表保护。
- VBA 宏或数字签名。
- 外部链接或外部数据连接。
- 数据透视表。
- 图表、图片、形状或嵌入文件。
- 当前 Excel 组件无法保证原样保留的其他特殊功能。

工具不得通过删除、扁平化或降级这些功能来强行生成文件。

### 第 15 轮：使用范围与数据源成本

已确认产品仅供个人低频使用：每周约 1 至 2 次，每次处理约 1 至 2 个 Excel，每个文件通常不超过 10 只股票。产品优先采用免费方案，不要求商业行情服务等级。

数据源专项评估见 [data-source-evaluation.md](data-source-evaluation.md)。

### 第 16 轮：首版行情数据源

已确认首版使用 AKShare 封装东方财富数据：

- 使用 `stock_zh_a_spot_em()` 获取沪深京当前上市 A 股清单及名称。
- 使用 `stock_zh_a_hist()` 获取指定日期日线，固定 `period="daily"`、`adjust=""`。
- 股票清单按自然日缓存，历史价格按任务查询。
- 不使用百度搜索、模拟浏览器访问同花顺或东方财富页面作为主方案或静默备用方案。
- 数据源失败时报告明确错误并停止处理，不生成文件。
- Excel 处理逻辑只依赖统一的 `MarketDataProvider`，不得直接依赖 AKShare 字段名称，以便未来替换数据源。

### 第 17 轮：文件处理与隐私

已确认 Excel 文件由后端临时处理：

- 客户端与后端之间强制使用 HTTPS。
- 上传文件保存到按任务隔离的临时目录，不写入业务数据库。
- 后端完成模板校验、行情查询、回填、自检后，以下载响应返回输出文件。
- 下载响应结束后立即删除输入和输出临时文件。
- 处理失败或连接异常时，临时文件最多保留 10 分钟，由清理任务删除。
- 不保存文件历史，不提供再次下载功能。
- 应用日志不得记录文件内容、股票交易数据或完整文件名。

### 第 18 轮：访问控制

已确认采用单用户访问密码：

- 不设置用户名、注册、找回密码和用户数据库。
- 密码由部署环境变量配置，服务端只使用安全哈希进行验证。
- 登录成功后通过 `HttpOnly`、`Secure`、`SameSite` Cookie 保存会话，默认有效期 7 天。
- 文件上传、行情查询和处理接口全部要求有效会话。
- 连续登录失败达到阈值后，暂时禁止继续尝试。
- 密码或会话信息不得出现在应用日志中。

### 第 19 轮：处理限制与进度

已确认以下运行规则：

- 单个上传文件最大 10 MB。
- 每次任务只处理一个 Excel 文件。
- 同一账号同时只允许执行一个任务。
- 股票代码不设置业务数量上限，但任务受 Excel 格式容量和 180 秒总超时限制。
- 行情请求发生网络超时、连接失败或服务端临时异常时，首次失败后最多重试 3 次，即最多尝试 4 次。
- 指定日期无行情、代码错误、模板不兼容等确定性错误不重试。
- 任务超过 180 秒后失败，不生成文件。

进度按以下阶段展示：

| 进度 | 阶段 |
| ---: | --- |
| 0%～10% | 文件上传 |
| 10%～25% | 格式、模板和时间段校验 |
| 25%～35% | 股票代码及上市状态校验 |
| 35%～80% | 查询股票名称和历史行情，按处理项数量递增 |
| 80%～90% | 写入允许的单元格 |
| 90%～98% | 输出文件自检 |
| 100% | 文件生成完成，可以下载 |

失败时进度停止，消息区域显示失败阶段、相关股票代码、日期和具体原因。

### 第 20 轮：自检与公式计算边界

已确认自检包含：

- 输出文件可以按原格式重新打开。
- 工作表数量、名称、顺序和显示状态不变。
- 历史时间段和非目标工作表未修改。
- E 至 S 的公式文本、单元格格式和合并关系未修改。
- 最新时间段日期标题与最终用户输入一致。
- A 列股票代码及顺序与输入一致。
- 已勾选的 B、C、D 与行情查询结果一致。
- 未勾选的 B、C、D 与处理前同坐标单元格值一致。

公式处理边界：

- 工具不计算或写入 E 至 S 的公式结果。
- 输入工作簿必须启用自动计算或打开时重新计算，否则判定为不兼容。
- 下载文件由 Excel/WPS 打开后重新计算公式。
- 自检不判断 E 至 S 收益结果是否正确。
- 公式缓存值不纳入受保护区域一致性校验，但公式文本和格式必须一致。
- `platform.xlsx` 已设置为打开时完整重新计算，符合要求。

### 第 21 轮：动态新增单元格格式

已确认输入数量超过现有股票数据行时，只动态扩展 A 至 D：

- 已存在的目标单元格只修改本次允许的值，保留原格式。
- 新增数据行的 A 至 D 样式复制自最新时间段最后一个标准股票数据行。
- 只复制字体、填充、边框、对齐和数字格式，不复制原值、公式、批注或超链接。
- A 列强制使用文本格式，保留六位代码前导零。
- B 列写入文本。
- C、D 写入数值，不转成文本，也不额外修改行情源返回的价格精度。
- 不设置整行行高，不向新增行的 E 至 S 复制任何格式、公式或数据。
- 找不到可靠的标准股票样板行时，模板不兼容并停止处理。

### 第 22 轮：错误分类与展示

已确认错误分为以下类别：

| 前缀 | 类别 |
| --- | --- |
| `AUTH` | 登录和会话错误 |
| `FILE` | 文件上传和基础格式错误 |
| `TEMPLATE` | 模板结构及 Excel 功能不兼容 |
| `INPUT` | 股票代码、日期和选项输入错误 |
| `MARKET` | 股票清单、名称和历史行情错误 |
| `OUTPUT` | Excel 写入、自检和下载文件生成错误 |
| `SYSTEM` | 超时、临时存储及其他系统错误 |

每条错误包含错误代码、失败阶段、简明标题、相关股票代码、日期、字段、原因和“未生成文件”结果说明。错误内容支持一键复制。

- 失败时进度条停在当前阶段并显示红色。
- 模板、代码等预检查错误尽量在一次处理中汇总列出。
- 多只股票缺少行情时，汇总列出全部股票、日期和字段。
- 网络错误完成最多 3 次重试后再报告最终失败。
- 页面不展示服务器路径、程序堆栈和接口原始响应。
- 任意错误均不生成部分结果文件。
- 服务端日志只记录任务编号、错误代码、阶段和耗时，不记录 Excel 内容。

### 第 23 轮：日期输入交互

已确认日期使用统一的“文本框 + 日历按钮”组合控件，不使用显示格式受操作系统影响的原生日期框：

- 页面始终显示并只接受严格的 `yyyy.MM.dd`。
- 上传后自动填入最新时间段的开始和结束日期。
- 用户可直接输入或通过日历选择；移动端直接输入时使用数字键盘。
- 日期不能为空且必须是真实有效的公历日期。
- 开始日期不得晚于结束日期。
- 修改后的开始日期和结束日期必须分别严格晚于上一时间段的对应日期。
- 输入不合法时立即显示字段错误并禁用“处理”按钮。
- 日期作为纯日期处理，不执行时区换算。
- 写回时间段标题时固定使用 `yyyy.MM.dd - yyyy.MM.dd`，短横线两侧各一个空格。

### 第 24 轮：页面布局与上传机制

已确认登录后使用单页响应式布局，顺序为：上传文件、股票代码多行输入、开始与结束日期、三个独立复选框、处理按钮、进度及消息区域。

- 上传后显示文件名、大小、格式、目标工作表和最新时间段。
- 股票代码使用多行文本框，并实时显示已识别数量。
- 桌面端日期并排、移动端上下排列。
- 输入无效时处理按钮禁用；处理期间锁定全部输入。
- 桌面端使用居中单列卡片，移动端使用全宽单列触控布局。

采用两次短暂上传：

1. 选择文件后上传分析副本，读取模板及日期后立即从后端删除。
2. 浏览器页面内存保留用户选择的原始文件对象。
3. 点击处理时重新上传同一文件，后端重新执行完整校验并处理。
4. 页面刷新或关闭后必须重新选择文件。

成功后自动触发下载；自动下载被浏览器阻止时显示“下载文件”按钮。下载内容只保存在当前页面内存，不存服务器历史。

### 第 25 轮：技术架构与部署

已确认采用单镜像 Docker 架构：

- 前端使用 Vue 3、TypeScript、Vite 和 PWA。
- Python FastAPI 负责单用户认证、任务编排、AKShare 行情、临时文件、进度和错误处理。
- Java 21 + Apache POI 作为本地 Excel 工作进程，同时处理 `.xls` 和 `.xlsx`。
- Python 通过受控本地子进程调用 Excel 引擎，不拆分为网络微服务。
- 进度使用 SSE 推送。
- 不使用数据库、Redis和消息队列，单用户任务状态保存在内存。
- PWA 只缓存前端静态资源，不缓存 Excel、行情响应和下载文件。
- 前端构建产物、Python 服务及 Java Excel 引擎打包为一个 Docker 镜像。
- 部署在普通 Linux Docker 主机，由部署平台或反向代理提供 HTTPS。
- 不采用纯 Serverless 部署。

详细架构见 [architecture.md](architecture.md)。

已确认冻结 MVP 范围：按第 5 节纳入项实施，明确排除通用 Excel 编辑、多用户、原生客户端、非目标证券、日期替代、受保护列处理和多行情源降级等能力。
